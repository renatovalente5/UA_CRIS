CIENCIAVITAE has two working API's, one in Quality Assurance(QA) and one in Production.

For now the team has access to the QA API through credentials given to Universidade de Aveiro.

This API presents some limitations, the biggest one being the fact that it only allows access to CIENCIA ID's that are also in QA environment, this means that a regular CIENCIA ID will have no results and will not exist in the QA API.

The documentation for both APIs can be found in:

  •         QA: https://qa.cienciavitae.pt/docs/
  •         Production: https://api.cienciavitae.pt/docs/

This API provides data both in XML and JSON.

PTCRISync is a framework that does the synchronization between a local CRIS and ORCID.

At the moment the api is not working, so the team downloaded the java library and with this built a webserver running the library and managed to import metadata of publications from one ORCID id.

The team created an API to use the PTCRIS library in .NET, it will be used to get the publications that aren't on RIA at the moment.

The documentation about the PTCRISync can be visited here: https://github.com/fccn/PTCRISync/wiki

The ORCID ID is a nonproprietary alphanumeric code to uniquely identify scientific and other academic authors and contributors.

This code relates to a profile in an international organization (ORCID) platform, containing data about works, education and employment oh each user.

We have succesfully used ORCID's public API, in order to retrieve information about researchers, which will be crucial to accomplish the main goal of our system.

However, we're still in the process of accessing the Member API, which will allow to publish or modify data of specific ORCID records, as a future goal.

euroCRIS is an international not-for-profit association that brings together experts on research information and CRIS’s.

The mission of euroCRIS is to promote cooperation among the research information community and interoperability of research information through CERIF, the Common European Research Information Format.

CERIF is the international standard data model for research information.

The primary objectives of the CERIF API are the following:

        To facilitate the interoperability of CRIS systems and their integration with other information systems.

        To enable the development of applications, services and components that can access and reuse information across different CRIS systems in a standard, unified way.

The CERIF data structure is defined as an entity-relationship model.

The CERIF standard also defines an XML representation of research information (CERIF XML) as a format for data exchange involving CRIS systems.

The HTTP response body contains XML divided into two parts:

        The header: which includes meta-information about the actual data returned.

        The payload: which contains the actual data retrieved from the CRIS system in response to the request.

Other areas of interest of euroCris also cover research databases, CRIS related data like scientific datasets, (open access) institutional repositories, as well as data access and exchange mechanisms, standards and guidelines and best practice for CRIS.

DSpace is a platform that allows the deposit of documents in any format (text, video, audio and data).

The DSpace system is mainly used as an institutional repository and plays three important roles:

        Facilitates the deposit and management of documents and their metadata;

        Facilitates access to documents through lists and searches;

        Facilitates long-term preservation of documents;

The RIA is the instutional repository of the University of Aveiro and uses DSpace to deposit articles, theses of researchers and professors and students.

RIA can be visited in the following website: https://ria.ua.pt/

SCOPUS is one of the largest abstracts and citation database of peer-reviewed literature.

SCOPUS has an API that allows its users to retrieve data about published documents, their authors and their respective bibliographic data.

Currently the team has access to the SCOPUS Subscriber API.

One of the most important pieces of information we can get from this API is details on the number of citations each published document has received (Citation Count, field: cited by).

The documentation for the API can be found in the following links:

        Information about the permitted metadata : https://dev.elsevier.com/academic_research_scopus.html

        General SCOPUS API information including response formats, most of them being JSON and XML : https://dev.elsevier.com/sc_apis.html

        Search API : https://dev.elsevier.com/documentation/SCOPUSSearchAPI.wadl

        Citations Count API : https://dev.elsevier.com/documentation/AbstractCitationCountAPI.wadl

        General CRIS integration information : https://dev.elsevier.com/ir_cris_vivo.html

        Elsevier APIs Settings : https://dev.elsevier.com/api_key_settings.html

        Multiple links of Elsevier APIs documentation : https://dev.elsevier.com/api_docs.html

Small example found online of python code that uses SCOPUS API to retrieve data: https://kitchingroup.cheme.cmu.edu/blog/2015/04/03/Getting-data-from-the-Scopus-API/
